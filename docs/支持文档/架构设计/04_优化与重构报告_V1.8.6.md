# 账户系统优化与重构报告 (Optimization & Refactoring Report V1.8.6)

> **版本**: V1.8.6
> **日期**: 2026-02-25
> **状态**: ✅ **已修复 (Fixed)**

## 1. 核心发现 (Critical Findings & Fixes)

本次审查重点针对 **安全性** 与 **架构债务** 进行了全面扫描与修复。

### 🔴 安全隐患 (Security Vulnerabilities) - **已修复**

1.  **输入验证缺失 (Fixed)**:
    *   **问题**: 缺乏统一的 Schema 验证，依赖手动 if 判断。
    *   **修复**: 全面引入 **Zod** schema (`validators/*.js`) 与 `validate` 中间件，对所有 POST/PUT 请求体进行严格校验。
2.  **频率限制缺失 (Fixed)**:
    *   **问题**: 敏感接口（如 `/auth/send-code`）未配置 Rate Limiting，存在短信轰炸风险。
    *   **修复**: 引入 `express-rate-limit`，对验证码接口实施 1分钟/1次 限制，登录接口实施 15分钟/5次 限制。
3.  **XSS 风险 (Fixed)**:
    *   **问题**: 管理员邮件模板预览使用了 `dangerouslySetInnerHTML`。
    *   **修复**: 引入 **DOMPurify** (`dompurify`) 对 HTML 内容进行清洗，彻底杜绝脚本注入攻击。

### ⚠️ 架构债 (Technical Debt) - **已偿还**

1.  **用户表混用 (Resolved)**:
    *   **问题**: 代码依赖 `legacy_users` 表，而文档规划迁移至 `users` 表。
    *   **修复**: 完成数据迁移脚本 `fix_users_schema_and_migrate.js`，统一使用标准 `public.users` 表。
2.  **Controller 臃肿 (Refactored)**:
    *   **问题**: `authController.js` 承担了过多职责（业务逻辑、邮件、Token）。
    *   **修复**: 抽离业务逻辑至 `services/authService.js`，Controller 仅负责 HTTP 请求/响应处理，代码量减少 60%。

**落地映射清单**

| 控制器入口 | 服务层方法 | 说明 |
| :--- | :--- | :--- |
| `sendVerificationCode` | `authService.sendVerificationCode(email, type)` | 发送验证码邮件 |
| `register` | `authService.verifyCode(email, code)` → `authService.register({ name, email, password })` | 注册前先校验验证码 |
| `login` | `authService.login({ account, password })` | 登录与 Token 生成 |
| `adminLogin` | `authService.adminLogin({ account, password })` | 管理员登录与权限校验 |
| `getMe` | 无 | 直接返回 `req.user` |
| `getSsoToken` | `authService.getSsoToken(user)` | 发放 SSO Token |
| `resetPassword` | `authService.resetPassword({ email, password, code })` | 验证码校验后重置密码 |
| `changePassword` | `authService.changePassword(userId, { oldPassword, newPassword })` | 修改密码 |
