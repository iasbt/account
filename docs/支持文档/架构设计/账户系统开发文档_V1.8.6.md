## 1. 核心架构：四合院项目矩阵 (The Quadrangle Matrix)

本系统 (Account System) 作为整个产品矩阵的 **数字基座**，负责统一身份认证 (SSO)、权限控制 (RBAC) 和数据中台服务。

### 1.1 拓扑结构 (Topology V1.8)

我们摒弃了过度设计的微服务架构，回归至最稳健的单体分层架构，但引入了 **Redis** 缓存与消息队列以支持高性能异步任务（如邮件服务）。同时在 V1.8.6 中进行了深度的 **安全加固** 与 **Service 层重构**。

## 2. 核心功能模块 (Core Modules)

### 2.1 统一身份认证 (SSO V2.0)
- **多应用支持**: 通过 `public.applications` 表动态管理接入应用（如 Gallery, Toolbox）。
- **Token 策略**:
    - **Standard**: 标准 JWT，用于内部微服务。
    - **Supabase**: 兼容 Supabase Auth 格式，用于 Gallery 等基于 Supabase 的应用。
- **流程**: 
    1. 应用跳转 `/api/auth/sso-token?target=...`
    2. 校验用户登录态
    3. 查询应用注册表与回调白名单
    4. 生成 Token 并重定向回 `target`

### 2.2 邮件服务 (Email Service V2.0)
- **架构**: 生产者-消费者模型 (BullMQ + Redis)。
- **特性**:
    - **可视化模板**: 支持 Mustache 语法 (`{{code}}`)，提供 HTML 代码编辑与实时预览。
    - **多服务商**: 支持 SMTP (QQ, Outlook, Resend) 动态切换与负载均衡。
    - **审计日志**: 记录每一封邮件的发送状态、耗时与错误信息。
    - **统计图表**: 24小时发送趋势图 (Recharts)。

### 2.3 管理员门户 (Admin Portal)
- **RBAC**: 基于 `is_admin` 字段的中间件拦截 (`middlewares/roleCheck.js`)。
- **功能**:
    - **用户管理**: 查看、编辑、删除用户。
    - **应用管理**: 一键接入新应用，生成 `app_id` 与 `secret`。
    - **系统监控**: 查看服务健康状态与版本信息。

### 2.4 安全加固 (Security Hardening V1.8.6)
- **输入验证 (Input Validation)**: 全面引入 **Zod** Schema，对所有 API 输入参数进行严格校验，拒绝非法请求。
- **频率限制 (Rate Limiting)**: 集成 `express-rate-limit`，针对不同接口实施差异化限流策略：
    - **全局**: 50次/15分钟
    - **验证码**: 3次/1分钟 (防止短信轰炸)
- **XSS 防护**: 管理后台引入 **DOMPurify**，对所有富文本内容进行清洗，彻底阻断 XSS 攻击。

## 3. 部署与运维 (Deployment & Operations)

### 3.1 自动化部署流水线
所有部署操作 **必须** 通过项目根目录下的 `deploy_remote.ps1` 脚本执行。

**执行命令**:
```powershell
.\deploy_remote.ps1 "v1.8.6: Security Hardening & Refactor"
```

**流水线步骤**:
1.  **Local**: 自动提交代码 (Git Commit & Push)。
2.  **Remote**: SSH 连接腾讯云，拉取最新代码 (Git Pull)。
3.  **Build**: 在服务器端构建 Docker 镜像 (`docker compose build`)。
4.  **Deploy**: 重启服务 (`down` -> `up -d`)。
5.  **Verify**: 自动执行 `/api/health` 检查，验证 Version 字段。

### 3.2 目录规范 (Directory Standard)
服务器端文件结构严格遵循：

```text
/home/ubuntu/account/
├── deploy/
│   └── correction/          <-- 部署核心目录 (Source of Truth)
│       ├── docker-compose.yml
│       ├── Dockerfile.api
│       ├── Dockerfile.web
│       ├── nginx.conf
│       └── servers.json     (PgAdmin Config)
├── server.js                <-- 后端入口
└── ...
```

## 4. 接口规范 (API Contract)
详细接口定义请参阅 `.trae/rules/05_API_Specs.md`。

### 4.1 健康检查 (Health Check)
*   **Endpoint**: `GET /api/health`
*   **Response**: `{"status":"ok", "service":"account-backend", "version":"1.8.5"}`

### 4.2 邮件服务 API (Admin Only)
*   `GET /api/admin/email/stats`: 获取 24h 发送趋势。
*   `GET /api/admin/email/logs`: 分页获取发送日志。
*   `PUT /api/admin/email/templates/:type`: 更新邮件模板。
