# Supabase 迁移至 JWT 指南

> **版本**: V2.0
> **状态**: Active
> **适用范围**: 所有曾使用 Supabase 兼容模式的应用
> **更新日期**: 2026-02-26

## 1. 概述 (Overview)

本指南说明如何将旧版应用（如 `gallery`）从 Supabase Auth 迁移到 Account 系统原生的 **纯 JWT 认证**。
旧版兼容模式（Legacy Mode）已彻底废弃。

## 2. 关键变更 (Key Changes)

| 特性 | 旧版 (Legacy) | 新版 (JWT V2.0) |
| :--- | :--- | :--- |
| **认证方式** | Token Injection (Cookie/Query) | Standard Bearer Token (Authorization Header) |
| **Token 格式** | Supabase JWT (含 `role: authenticated`) | Standard JWT (RFC 7519) |
| **SSO 流程** | `/api/auth/sso-token` (JSON Response) | `/api/sso/issue` (Redirect Flow) |
| **用户表** | `auth.users` (Supabase Schema) | `public.users` (Standard Schema) |
| **应用注册** | `config/allowlist.js` (硬编码) | `public.applications` (数据库) |

## 3. 迁移步骤 (Migration Steps)

### 3.1 数据库迁移
1.  **备份数据**: 导出 `public.users` 表。
2.  **注册应用**: 在 `public.applications` 表中插入应用记录，生成 `secret`。
    ```sql
    INSERT INTO public.applications (app_id, name, secret, allowed_origins, is_active)
    VALUES ('gallery', 'Gallery App', 'NEW_SECRET...', ARRAY['https://gallery.iasbt.com'], true);
    ```

### 3.2 后端代码迁移
1.  **移除 Supabase 依赖**: 删除所有 `supabase-js` 或 `gotrue-js` 引用。
2.  **更新 JWT 验证**: 使用 `jsonwebtoken` 库替换 Supabase 的验证逻辑。
    ```javascript
    // OLD (Supabase)
    const { user, error } = await supabase.auth.api.getUser(token);

    // NEW (JWT)
    const decoded = jwt.verify(token, NEW_SECRET);
    ```
3.  **更新中间件**: 确保 API 网关或后端服务能正确解析 `Authorization: Bearer <token>` 头。

### 3.3 前端代码迁移
1.  **更新登录跳转**: 将登录链接指向 `https://account.iasbt.com/sso/issue?target=...`。
2.  **更新 Token 解析**: 从 URL Hash 中解析 `access_token`，不再依赖 Cookie 或 Query 参数。
3.  **更新 API 请求**: 在所有请求头中添加 `Authorization`。
    ```javascript
    // Axios 示例
    axios.defaults.headers.common['Authorization'] = `Bearer ${accessToken}`;
    ```

## 4. 常见问题 (FAQ)

### Q1: 旧版 Token 还能用吗？
**不能**。旧版 Token 已失效，用户必须重新登录以获取新格式 Token。

### Q2: 用户需要重置密码吗？
**不需要**。用户密码哈希存储在 `public.users` 表中，与认证方式无关。

### Q3: 如何处理 RLS (Row Level Security)？
若使用 PostgREST，需更新数据库角色权限，确保 `current_setting('request.jwt.claim.sub')` 能正确映射到 `public.users.id`。
