# Supabase 迁移至 OAuth 2.0 指南

> **版本**: V2.1
> **状态**: Active
> **适用范围**: 所有曾使用 Supabase 兼容模式或旧版 JWT 模式的应用
> **更新日期**: 2026-02-27

## 1. 概述

本指南说明如何将旧版应用（如 `gallery`）从 Supabase Auth 或旧版 JWT 模式迁移到 Account 系统原生的 **OAuth 2.0 (PKCE) + JWKS** 认证体系。
旧版兼容模式（Legacy Mode）与隐式流（Implicit Flow）已彻底废弃。

## 2. 关键变更

| 特性 | 旧版 (Legacy/V1.8) | 新版 (OAuth 2.0 V2.1) |
| :--- | :--- | :--- |
| **认证协议** | Token Injection / Implicit Flow | OAuth 2.0 Authorization Code + PKCE |
| **Token 签名** | HS256 (共享 Secret) | RS256 (非对称加密) |
| **公钥获取** | 手动配置 Secret | 自动发现 (JWKS Endpoint) |
| **Token 获取** | URL Hash (`#access_token=...`) | POST `/api/oauth/token` |
| **SSO 入口** | `/api/sso/issue` (已移除) | `/api/oauth/authorize` |
| **黑名单** | Redis Key (Stateful) | Refresh Token Rotation (Stateless) |

## 3. 迁移步骤

### 3.1 数据库迁移
1.  **应用注册**: 确保应用在 `public.applications` 表中注册，并配置正确的 `allowed_origins`（严格匹配）。
    ```sql
    -- 示例：更新 Gallery 回调地址
    UPDATE public.applications 
    SET allowed_origins = ARRAY['http://localhost:5173/auth/callback', 'https://gallery.iasbt.com/auth/callback']
    WHERE app_id = 'gallery';
    ```

### 3.2 后端代码迁移 (Node.js/Go/Python)
1.  **移除 Secret**: 不再需要配置 `JWT_SECRET`。
2.  **引入 JWKS**: 使用 `jwks-rsa` 或类似库动态获取公钥。
    ```javascript
    // Node.js (express-jwt) 示例
    const { expressjwt: jwt } = require("express-jwt");
    const jwksRsa = require("jwks-rsa");

    app.use(jwt({
      secret: jwksRsa.expressJwtSecret({
        cache: true,
        rateLimit: true,
        jwksRequestsPerMinute: 5,
        jwksUri: "https://account.iasbt.com/.well-known/jwks.json"
      }),
      algorithms: ["RS256"]
    }));
    ```

### 3.3 前端代码迁移 (React/Vue)
1.  **生成 PKCE**: 登录前生成 `code_verifier` 和 `code_challenge`。
2.  **构建授权链接**:
    ```javascript
    const authUrl = `https://account.iasbt.com/api/oauth/authorize?response_type=code&client_id=gallery&redirect_uri=${callbackUrl}&code_challenge=${challenge}&code_challenge_method=S256`;
    window.location.href = authUrl;
    ```
3.  **处理回调**: 在 `/auth/callback` 路由获取 `code`，调用后端（或直接调用 Account API，若是 Public Client）换取 Token。
    ```javascript
    // 换取 Token
    const res = await axios.post('https://account.iasbt.com/api/oauth/token', {
      grant_type: 'authorization_code',
      code,
      client_id: 'gallery',
      redirect_uri: callbackUrl,
      code_verifier: verifier
    });
    const { access_token, refresh_token } = res.data;
    ```

## 4. 常见问题 (FAQ)

### Q1: 为什么 `/api/sso/issue` 返回 410 Gone？
该接口因安全原因已永久下线。请按照本指南迁移至 OAuth 2.0 流程。

### Q2: 什么是 Refresh Token Rotation？
每次刷新 Token 时，旧的 Refresh Token 会立即失效，并颁发一个新的 Refresh Token。这能有效防止令牌被盗用后的持续访问。

### Q3: 如何验证 RS256 签名？
你需要从 `/.well-known/jwks.json` 获取公钥。大多数 JWT 库（如 `jsonwebtoken` + `jwks-rsa`）都支持自动处理。
