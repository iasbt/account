# SSO 单点登录对接指南

> **版本**: V2.1
> **状态**: Active
> **适用范围**: 所有接入 Account 系统的客户端与服务端
> **更新日期**: 2026-03-07

## 1. 概述 (Overview)

本指南详细说明如何将子应用（如 Gallery, Toolbox, LifeOS）接入 Account 系统的 SSO（单点登录）服务。
SSO 采用标准的 **OAuth 2.0 授权码模式 (Authorization Code Flow) + PKCE**，并使用 **RS256** 非对称加密签名。

**⚠️ 重要提示**:
*   旧版 `Redirect-Based` (URL Hash Fragment) 模式已废弃。
*   旧版 `Supabase` 兼容模式已废弃。
*   所有新接入应用**必须**实现 PKCE 流程。

## 2. 准备工作 (Preparation)

### 2.1 注册应用 (Application Registration)
所有接入 SSO 的应用必须在数据库 `public.applications` 表中注册。

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `app_id` | `text` | 唯一标识符 (如 `gallery`)，作为 OAuth `client_id` |
| `name` | `text` | 应用显示名称 |
| `secret` | `text` | **Client Secret** (仅后端应用需要，SPA 可不使用或慎用) |
| `allowed_origins` | `text[]` | 允许的 `redirect_uri` 列表 (必须严格匹配) |
| `is_active` | `boolean` | 是否启用 |

#### 注册示例 (SQL)
```sql
INSERT INTO public.applications (app_id, name, secret, allowed_origins, is_active)
VALUES (
  'gallery',
  'Gallery App',
  'your-secure-random-secret-key-32-chars',
  ARRAY['https://iasbt.cloud/auth/callback', 'http://localhost:5173/auth/callback'],
  true
);
```

## 3. 对接流程 (Integration Flow)

### 步骤 1: 生成 PKCE 挑战 (Client)
客户端生成 `code_verifier` (随机字符串) 并计算 `code_challenge` (SHA-256 哈希)。

```typescript
// 生成示例 (伪代码)
const codeVerifier = generateRandomString(64);
const codeChallenge = base64UrlEncode(sha256(codeVerifier));
// 存储 codeVerifier 到本地 (如 sessionStorage) 以便后续使用
sessionStorage.setItem('pkce_verifier', codeVerifier);
```

### 步骤 2: 发起授权请求 (Authorization Request)
引导用户访问 Account 授权端点：

```
GET https://account.iasbt.cloud/api/oauth/authorize
  ?client_id=gallery
  &redirect_uri=https://iasbt.cloud/auth/callback
  &response_type=code
  &scope=openid profile email
  &state=xyz123
  &code_challenge=xxxxxx
  &code_challenge_method=S256
```

### 步骤 3: 用户认证与重定向
*   用户在 Account 登录（如未登录）。
*   Account 验证通过后，生成 Authorization Code。
*   重定向回 `redirect_uri`，附带 `code` 和 `state`：

```
https://iasbt.cloud/auth/callback?code=SplxlOBeZQQYbYS6WxSbIA&state=xyz123
```

### 步骤 4: 换取令牌 (Token Exchange)
客户端（或后端）使用 Code 换取 Access Token。必须携带 `code_verifier`。

**请求**:
```http
POST /api/oauth/token
Content-Type: application/json

{
  "grant_type": "authorization_code",
  "code": "SplxlOBeZQQYbYS6WxSbIA",
  "redirect_uri": "https://iasbt.cloud/auth/callback",
  "client_id": "gallery",
  "code_verifier": "之前生成的verifier",
  "client_secret": "如果是后端应用则必填"
}
```

**响应**:
```json
{
  "access_token": "eyJhbGciOiJSUzI1NiIs...",
  "refresh_token": "def50200...",
  "token_type": "Bearer",
  "expires_in": 900,
  "scope": "profile"
}
```

### 步骤 5: 验证 Token (Validation)
使用 Account 提供的公钥验证 Token 签名 (RS256)。

*   **JWKS 端点**: `https://account.iasbt.cloud/.well-known/jwks.json`
*   **Issuer**: `https://account.iasbt.cloud`
*   **Audience**: `gallery`

## 4. 刷新令牌 (Refresh Token)

Access Token 过期后，使用 Refresh Token 获取新令牌。

**请求**:
```http
POST /api/oauth/token
Content-Type: application/json

{
  "grant_type": "refresh_token",
  "refresh_token": "之前的refresh_token",
  "client_id": "gallery",
  "client_secret": "..."
}
```

**响应**:
返回新的 `access_token` 和 **新的** `refresh_token` (旧的立即失效)。
