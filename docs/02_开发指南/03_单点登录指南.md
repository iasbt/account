# SSO 单点登录对接指南

> **版本**: V2.0
> **状态**: Active
> **适用范围**: 所有接入 Account 系统的客户端与服务端
> **更新日期**: 2026-02-26

## 1. 概述 (Overview)

本指南详细说明如何将子应用（如 Gallery、Toolbox、LifeOS）接入 Account 系统的 SSO（单点登录）服务。
SSO 采用 **Redirect-Based** 模式，通过 URL Hash Fragment 传递 Token。

**⚠️ 重要提示**: 旧版兼容 Supabase 的 SSO 模式（Token Injection）已废弃。

## 2. 准备工作 (Preparation)

### 2.1 注册应用 (Application Registration)
所有接入 SSO 的应用必须在数据库 `public.applications` 表中注册。

| 字段 | 类型 | 说明 |
| :--- | :--- | :--- |
| `app_id` | `text` | 唯一标识符 (如 `gallery`) |
| `name` | `text` | 应用显示名称 |
| `secret` | `text` | **JWT 签名密钥** (推荐每个应用独立生成) |
| `allowed_origins` | `text[]` | 允许的 Origin 列表 (用于 CORS 和 SSO 匹配) |
| `is_active` | `boolean` | 是否启用 |

#### 注册示例 (SQL)
```sql
INSERT INTO public.applications (app_id, name, secret, allowed_origins, is_active)
VALUES (
  'gallery',
  'Gallery App',
  'your-secure-random-secret-key-32-chars', 
  ARRAY['https://gallery.iasbt.com', 'http://localhost:5173'],
  true
);
```

## 3. 对接流程 (Integration Flow)

### 步骤 1: 发起登录
子应用引导未登录用户访问 Account SSO 发放页：
```
GET https://account.iasbt.com/sso/issue?target=https://gallery.iasbt.com/auth/callback
```
*   `target`: 子应用接收 Token 的回调地址 (必须与 `allowed_origins` 匹配)。

### 步骤 2: 用户认证
*   若用户在 Account 未登录，将跳转至 `/login`。
*   登录成功后，Account 自动重定向回 `target` URL。

### 步骤 3: 接收 Token
Account 将 Token 以 **URL Hash Fragment** 形式附加在 `target` 后：
```
https://gallery.iasbt.com/auth/callback#access_token=eyJ...&refresh_token=eyJ...&expires_in=3600&token_type=bearer
```

### 步骤 4: 解析与验证 (前端)
子应用前端解析 Hash，获取 `access_token` 并存储。

#### 前端解析示例 (JavaScript)
```javascript
const hash = window.location.hash.substring(1);
const params = new URLSearchParams(hash);
const accessToken = params.get('access_token');
const refreshToken = params.get('refresh_token');

if (accessToken) {
  localStorage.setItem('access_token', accessToken);
  window.location.hash = ''; // 清除 Hash
} else {
  console.error('SSO 登录失败');
}
```

### 步骤 5: 验证 Token (后端)
后端（或 API 网关）使用 **共享的 Secret** 验证 JWT 签名。

#### 后端验证示例 (Node.js)
```javascript
import jwt from 'jsonwebtoken';

const APP_SECRET = 'your-secure-random-secret-key-32-chars'; // 与数据库一致

function verifyUserToken(token) {
  try {
    const decoded = jwt.verify(token, APP_SECRET, {
      algorithms: ['HS256'],
      issuer: 'account.iasbt.com',
      audience: 'gallery' // 你的 app_id
    });
    return decoded;
  } catch (err) {
    console.error('Invalid Token:', err.message);
    return null;
  }
}
```

## 4. 常见问题 (FAQ)

### Q1: 如何处理 Token 过期？
使用 `refresh_token` 向 `/api/auth/refresh` 换取新 Token（待实现）。目前 Access Token 有效期较短，Refresh Token 有效期较长。

### Q2: 本地开发如何调试？
确保 `allowed_origins` 包含 `http://localhost:PORT`。

### Q3: 如何获取 Secret？
请联系管理员查询 `public.applications` 表。
