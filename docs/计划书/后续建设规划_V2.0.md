## 2. Phase 2: 相册集成 (Album Integration)
**核心目标**: 实现高性能、低成本的图片存储与分发服务。

### 2.1 技术选型
*   **存储**: **Cloudflare R2** (S3 兼容，零出口流量费)。
*   **处理**: **Sharp** (Node.js 高性能图片处理库，用于缩略图生成/格式转换)。
*   **协议**: AWS SDK v3 (`@aws-sdk/client-s3`)。

### 2.2 详细实施计划

#### Step 1: 基础设施 (Infrastructure)
*   [ ] **配置 R2**: 获取 Account ID, Access Key, Secret Key, Bucket Name。
*   [ ] **环境变量**: 更新 `.env` (本地) 与 Portainer (生产)。
    ```bash
    R2_ACCOUNT_ID=...
    R2_ACCESS_KEY_ID=...
    R2_SECRET_ACCESS_KEY=...
    R2_BUCKET_NAME=...
    R2_PUBLIC_DOMAIN=...
    ```

#### Step 2: 后端开发 (Backend)
*   [ ] **依赖安装**: `npm install @aws-sdk/client-s3 sharp multer`。
*   [ ] **Service 层**: 创建 `services/storageService.js` (封装 S3 Upload/Delete/List)。
*   [ ] **Controller 层**: 创建 `controllers/albumController.js`。
    *   `POST /api/album/upload`: 接收文件 -> Sharp 压缩/WebP 转换 -> 上传 R2 -> 写入 DB。
    *   `GET /api/album`: 分页获取图片列表。
    *   `DELETE /api/album/:id`: 软删除或物理删除。
*   [ ] **Database**: 设计 `photos` 表 (id, user_id, url, thumbnail_url, meta, created_at)。

#### Step 3: 前端开发 (Frontend)
*   [ ] **UI 组件**: 开发瀑布流图片展示组件 (`MasonryLayout`)。
*   [ ] **上传组件**: 支持拖拽上传、进度显示、客户端压缩预览。
*   [ ] **页面**: `/album` 路由。

## 3. Phase 3: SSO 生态扩展 (Ecosystem)
**核心目标**: 确立 Account 系统作为所有子应用 (Gallery, Toolbox, LifeOS) 的唯一登录入口。

### 3.1 架构设计
基于 **CAS (Central Authentication Service)** 模式，使用 `Application Registry` 控制权限。

### 3.2 详细实施计划

#### Step 1: 注册表配置 (Registry)
*   [ ] **Toolbox 接入**:
    *   在 `applications` 表中注册 Toolbox。
    *   配置 `allowed_origins` (如 `https://toolbox.iasbt.com`)。
    *   生成并分发 `SSO_SECRET_TOOLBOX`。
*   [ ] **Life OS 接入**:
    *   注册 Life OS。
    *   配置 `allowed_origins`。

#### Step 2: 跨域验证 (Cross-Domain Verification)
*   [ ] **模拟测试**: 创建 `tests/sso-client-demo.html`。
    *   模拟子应用接收 `access_token` (Hash Fragment)。
    *   验证 Token 签名有效性。
*   [ ] **E2E 流程**:
    *   用户访问 Toolbox -> 重定向至 Account -> 登录 -> 重定向回 Toolbox (带 Token)。

#### Step 3: 集成指南 (Documentation)
*   [ ] 更新 `docs/支持文档/指南手册/Account_Gallery_SSO集成指南.md`，增加通用接入部分。

## 4. 资源需求与风险

### 4.1 资源
*   **R2 存储桶**: 需提前在 Cloudflare 控制台创建。
*   **开发时间**: 预计 2 周 (10 Man-Days)。

### 4.2 风险
*   **图片处理性能**: Node.js 处理大图可能阻塞 Event Loop -> 解决方案：使用 `worker_threads` 或限制上传大小。
*   **流量滥用**: R2 公开链接可能被盗链 -> 解决方案：配置 Cloudflare WAF 或使用 Signed URL (Phase 2.x)。

---

## 5. 执行顺序建议
建议优先执行 **Phase 3 (SSO)**，因为它是纯逻辑工作，且能快速串联生态；随后执行 **Phase 2 (Album)**，因为涉及外部存储服务对接，复杂度较高。
