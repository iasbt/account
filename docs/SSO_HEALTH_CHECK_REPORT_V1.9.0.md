### 1.1 完整的 PKCE 支持 (Full PKCE Support)
*   **状态**: ✅ **已启用**
*   **机制**: 强制要求授权码流程使用 `code_challenge` 和 `code_verifier`。
*   **效果**: 彻底消除了授权码截获攻击的风险，即使在不安全的通信信道中也能保证安全。

*   **效果**: 杜绝了子域名劫持或 URL 参数绕过漏洞。

## 2. 架构升级 (Architecture Upgrades)

### 2.1 RS256 非对称加密 (Asymmetric Encryption)
*   **状态**: ✅ **已迁移**
*   **机制**:
    *   **私钥**: 仅由 Account 服务持有，用于签名。
    *   **公钥**: 通过 `/.well-known/jwks.json` 公开，用于验证。
*   **优势**: 客户端（如 Gallery）无需保存敏感的 Secret，降低了密钥泄露导致系统沦陷的风险。

### 2.2 刷新令牌轮换 (Refresh Token Rotation)
*   **状态**: ✅ **已实施**
*   **机制**:
    *   **Access Token**: 短效 (1小时)。
    *   **Refresh Token**: 长效 (7天)，每次使用后立即失效并颁发新令牌。
    *   **重放检测**: 若旧 Refresh Token 被重用，系统自动吊销该用户所有令牌。
*   **优势**: 替代了 Redis 黑名单机制，解决了性能瓶颈，同时提供了更高的安全性（令牌盗用检测）。
