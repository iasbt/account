# 架构设计说明书

> **评估对象**: Account System V2.1
> **版本**: V2.1
> **状态**: ✅ **已实施**

## 1. 架构概览

### 1.1 逻辑架构
系统采用 **分层架构**，严格分离关注点。

*   **前端**:
    *   **React 19 + Vite**: 位于 `src/`。
    *   **Zustand**: 状态管理 (`useAuthStore`)。
    *   **管理后台**: 集成在 `/admin` 路由下。
*   **后端**:
    *   **Express**: 核心 Web 框架。
    *   **控制器**: 处理 HTTP 请求/响应 (`controllers/*.js`)。
    *   **服务层**: 核心业务逻辑 (`services/*.js`)。
    *   **校验器**: 输入验证 (`validators/*.js`)。
    *   **中间件**: 切面逻辑 (`middlewares/*.js`)。
*   **数据层**:
    *   **PostgreSQL**: 关系型数据存储。
    *   **Redis**: 缓存与消息队列存储（仅用于邮件队列）。

### 1.2 部署架构
基于 **Docker Compose** 的容器化部署，运行于 `correction_default` 网络。

*   **account-backend**: Node.js 20 容器，承载 API 与 Email Worker。
*   **account-frontend**: Nginx 容器，承载静态资源与反向代理。
*   **iasbt-postgres**: PostgreSQL 14 数据库。
*   **redis**: Redis 7 缓存/队列。
*   **pgadmin**: 数据库可视化管理。

### 1.3 关键技术栈
*   **语言**: JavaScript (Node.js) / TypeScript (React)
*   **API**: RESTful / OAuth 2.0 / OIDC Discovery
*   **队列**: BullMQ (基于 Redis)
*   **安全**: Helmet, CORS, RS256 (JWT), PKCE, Refresh Token Rotation

## 2. 核心模块设计

### 2.1 邮件服务（V2）
采用 **异步队列模式** 设计：
1.  **生产者**: API 接收发送请求，推入 Redis 队列。
2.  **消费者**: 后台 Worker 监听队列，通过 `nodemailer` 发送。
3.  **配置**: 支持多 SMTP 服务商动态切换 (`email_providers` 表)。
4.  **模板**: 支持 HTML 模板动态渲染 (`email_templates` 表)。

### 2.2 多应用 SSO (V2.1)
基于 **OAuth 2.0 授权码模式** 的安全 SSO：
1.  **注册表**: 数据库存储 AppID、Secret、Allowed Origins。
2.  **流程**: Authorize (PKCE) -> Code -> Token (RS256) -> Refresh。
3.  **安全**: 强制 PKCE，移除隐式流，支持 JWKS 验签。
4.  **轮换**: 刷新令牌轮换机制（Refresh Token Rotation）。

### 2.3 管理员门户
集成式管理后台：
*   **认证**: 独立管理员登录（`is_admin` 校验）。
*   **功能**: 用户管理、应用管理、邮件日志查看、服务商配置。

### 2.4 认证模块路由与职责拆分

| 路由入口 | 核心处理 | 说明 |
| :--- | :--- | :--- |
| `/oauth/authorize` | `router.use(oidcProvider.callback())` | OAuth 2.0 授权端点 (PKCE，支持 GET/POST) |
| `/oauth/token` | `router.use(oidcProvider.callback())` | 换取 Access/Refresh Token |
| `register` | `authService.verifyCode` → `authService.register` | 注册前先校验验证码 |
| `login` | `authService.login` | 登录与 Token 生成 |
| `getMe` | 无 | 直接返回 `req.user` (验证 JWT) |
| `/.well-known/*` | `router.use(oidcProvider.callback())` | OIDC 发现文档与 JWKS 公钥 |

## 3. 目录结构规范

```text
/
├── controllers/   # 业务逻辑控制器
├── routes/        # 路由定义
├── middlewares/   # 中间件
├── config/        # 配置文件
├── utils/         # 工具函数
├── deploy/        # 部署脚本与配置
├── docs/          # 项目文档
│   ├── 01_架构设计 ~ 04_部署运维/  # 现行有效文档
│   └── 归档中心/  # 历史归档
└── src/           # 前端源码
```

## 4. 演进历史

*   **V1.0 - V1.5**: 单体架构，`server.js` 包含所有逻辑。
*   **V1.6**: 引入分层架构，拆分控制器与路由。
*   **V1.8**: 引入 Redis 与 BullMQ，实现异步邮件服务与可视化应用管理。
*   **V2.1**: 引入 OAuth 2.0 (PKCE) 与 Refresh Token Rotation，全面升级安全性。
