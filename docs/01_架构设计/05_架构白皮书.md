# 架构设计说明书

> **评估对象**: Account System V1.8.6
> **版本**: V2.1
> **状态**: ✅ **已实施**

## 1. 架构概览

### 1.1 逻辑架构
系统采用 **分层架构**，严格分离关注点。

*   **前端**:
    *   **React 19 + Vite**: 位于 `src/`。
    *   **Zustand**: 状态管理 (`useAuthStore`)。
    *   **管理后台**: 集成在 `/admin` 路由下。
*   **后端**:
    *   **Express**: 核心 Web 框架。
    *   **控制器**: 处理 HTTP 请求/响应 (`controllers/*.js`)。
    *   **服务层**: 核心业务逻辑 (`services/*.js`)（V1.8.6 起引入）。
    *   **校验器**: 输入验证 (`validators/*.js`)（V1.8.6 起引入）。
    *   **中间件**: 切面逻辑 (`middlewares/*.js`)。
*   **数据层**:
    *   **PostgreSQL**: 关系型数据存储。
    *   **Redis**: 缓存与消息队列存储。

### 1.2 部署架构
基于 **Docker Compose** 的容器化部署，运行于 `correction_default` 网络。

*   **account-backend**: Node.js 20 容器，承载 API 与 Email Worker。
*   **account-frontend**: Nginx 容器，承载静态资源与反向代理。
*   **iasbt-postgres**: PostgreSQL 14 数据库。
*   **redis**: Redis 7 缓存/队列。
*   **pgadmin**: 数据库可视化管理。

### 1.3 关键技术栈
*   **语言**: JavaScript (Node.js) / TypeScript (React)
*   **API**: RESTful
*   **队列**: BullMQ (基于 Redis)
*   **安全**: Helmet, CORS, Bcrypt, JWT

## 2. 核心模块设计

### 2.1 邮件服务（V2）
采用 **异步队列模式** 设计：
1.  **生产者**: API 接收发送请求，推入 Redis 队列。
2.  **消费者**: 后台 Worker 监听队列，通过 `nodemailer` 发送。
3.  **配置**: 支持多 SMTP 服务商动态切换 (`email_providers` 表)。
4.  **模板**: 支持 HTML 模板动态渲染 (`email_templates` 表)。

### 2.2 多应用 SSO
基于 **数据库注册表 (`applications`)** 的动态 SSO：
1.  **注册表**: 数据库存储 AppID、Secret、Allowed Origins。
2.  **流程**: Redirect -> Issue Token -> Callback。
3.  **兼容**: 支持 Supabase JWT 与 Standard JWT 两种模式。

### 2.3 管理员门户
集成式管理后台：
*   **认证**: 独立管理员登录（`is_active` 校验）。
*   **功能**: 用户管理、应用管理、邮件日志查看、服务商配置。

### 2.4 认证模块职责拆分

| 控制器入口 | 服务层方法 | 说明 |
| :--- | :--- | :--- |
| `sendVerificationCode` | `authService.sendVerificationCode(email, type)` | 发送验证码邮件 |
| `register` | `authService.verifyCode(email, code)` → `authService.register({ name, email, password })` | 注册前先校验验证码 |
| `login` | `authService.login({ account, password })` | 登录与 Token 生成 |
| `adminLogin` | `authService.adminLogin({ account, password })` | 管理员登录与权限校验 |
| `getMe` | 无 | 直接返回 `req.user` |
| `getSsoToken` | `authService.getSsoToken(user)` | 发放 SSO Token |
| `resetPassword` | `authService.resetPassword({ email, password, code })` | 验证码校验后重置密码 |
| `changePassword` | `authService.changePassword(userId, { oldPassword, newPassword })` | 修改密码 |

## 3. 目录结构规范

```text
/
├── controllers/   # 业务逻辑控制器
├── routes/        # 路由定义
├── middlewares/   # 中间件
├── config/        # 配置文件
├── utils/         # 工具函数
├── deploy/        # 部署脚本与配置
├── docs/          # 项目文档
│   ├── 支持文档/  # 现行有效文档
│   └── 归档中心/  # 历史归档
└── src/           # 前端源码
```

## 4. 演进历史

*   **V1.0 - V1.5**: 单体架构，`server.js` 包含所有逻辑。
*   **V1.6**: 引入分层架构，拆分控制器与路由。
*   **V1.8**: 引入 Redis 与 BullMQ，实现异步邮件服务与可视化应用管理。
