# 数据库架构文档 (Database Architecture)

> **版本 (Version)**: V2.1
> **生效日期 (Effective Date)**: 2026-02-27
> **状态 (Status)**: Active
> **变更 (Changes)**: Added OAuth 2.0 tables (`oauth_codes`, `refresh_tokens`)

## 1. 概述

Account 系统作为中心化身份验证服务，管理着用户身份、应用接入注册表以及邮件服务配置。数据库采用 **PostgreSQL 14**。

## 2. 模式

*   `public`: 核心账户、应用注册表、邮件服务、OAuth 2.0 数据。
*   `gallery`: 图库子应用数据（预留 Schema）。

## 3. 核心表结构

### 3.1 用户表 (`users`)
*说明：此表为标准用户表，使用 TEXT 形式的 UUID 作为主键。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `TEXT` | 用户 ID (UUID) | PK |
| `name` | `VARCHAR(255)` | 用户名 | NOT NULL |
| `email` | `VARCHAR(255)` | 邮箱地址 | UNIQUE, NOT NULL |
| `password` | `VARCHAR(255)` | Bcrypt 哈希密码 | NOT NULL |
| `avatar_url` | `VARCHAR(255)` | 头像链接 | |
| `is_admin` | `BOOLEAN` | 是否管理员 | DEFAULT FALSE |
| `created_at` | `TIMESTAMP` | 创建时间 | DEFAULT NOW() |
| `updated_at` | `TIMESTAMP` | 更新时间 | DEFAULT NOW() |

### 3.2 遗留用户表 (`legacy_users`)
*说明：当前管理端用户管理仍在读取此表，未可直接删除。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | 用户 ID | PK |
| `username` | `VARCHAR(255)` | 用户名 | UNIQUE, NOT NULL |
| `email` | `VARCHAR(255)` | 邮箱 | UNIQUE, NOT NULL |
| `password_hash` | `VARCHAR(255)` | 密码哈希 | NOT NULL |
| `is_admin` | `BOOLEAN` | 是否管理员 | DEFAULT FALSE |
| `created_at` | `TIMESTAMPTZ` | 创建时间 | DEFAULT NOW() |
| `updated_at` | `TIMESTAMPTZ` | 更新时间 | DEFAULT NOW() |

### 3.3 应用注册表（SSO）(`applications`)
*用于管理接入 Account SSO 的子应用。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | 内部 ID | PK, DEFAULT gen_random_uuid() |
| `app_id` | `VARCHAR` | 应用唯一标识 (如 `gallery`) | UNIQUE, NOT NULL |
| `name` | `VARCHAR` | 应用显示名称 | NOT NULL |
| `allowed_origins` | `TEXT[]` | 允许的回调域名列表 | NOT NULL |
| `secret` | `VARCHAR` | 应用专属签名密钥 | NULLABLE (Public Clients) |
| `is_active` | `BOOLEAN` | 启用状态 | DEFAULT TRUE |
| `created_at` | `TIMESTAMP` | 创建时间 | DEFAULT NOW() |
| `updated_at` | `TIMESTAMP` | 更新时间 | DEFAULT NOW() |

### 3.4 OAuth 授权码表 (`oauth_codes`) (新增 V2.1)
*用于存储 OAuth 2.0 授权码模式（Authorization Code Flow）的临时授权码。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `code` | `VARCHAR(255)` | 授权码 | PK |
| `client_id` | `VARCHAR(50)` | 应用 ID | FK -> applications.app_id |
| `user_id` | `TEXT` | 用户 ID | FK -> users.id |
| `redirect_uri` | `TEXT` | 回调地址 | NOT NULL |
| `scope` | `TEXT` | 授权范围 | DEFAULT 'profile' |
| `expires_at` | `TIMESTAMPTZ` | 过期时间 | NOT NULL (通常 10分钟) |
| `used` | `BOOLEAN` | 是否已使用 | DEFAULT FALSE |
| `code_challenge` | `TEXT` | PKCE 挑战码 | NULLABLE |
| `code_challenge_method` | `VARCHAR(10)` | PKCE 方法 | DEFAULT 'S256' |
| `created_at` | `TIMESTAMPTZ` | 创建时间 | DEFAULT NOW() |

### 3.5 刷新令牌表 (`refresh_tokens`) (新增 V2.1)
*用于实现刷新令牌轮换（Refresh Token Rotation）机制，替代 Redis 黑名单。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | 令牌 ID | PK, DEFAULT gen_random_uuid() |
| `user_id` | `TEXT` | 用户 ID | FK -> users.id |
| `token_hash` | `TEXT` | 令牌哈希值 | NOT NULL, INDEX |
| `expires_at` | `TIMESTAMPTZ` | 过期时间 | NOT NULL (当前默认 14天) |
| `revoked` | `BOOLEAN` | 是否撤销 | DEFAULT FALSE |
| `replaced_by_token_hash` | `TEXT` | 轮换后的新令牌哈希 | 用于检测盗用 |
| `client_id` | `TEXT` | 客户端 ID | OPTIONAL |
| `created_at` | `TIMESTAMPTZ` | 创建时间 | DEFAULT NOW() |

### 3.6 邮件服务提供商 (`email_providers`)
*支持多 SMTP 服务商配置。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | ID | PK |
| `name` | `VARCHAR` | 提供商名称 (如 `Outlook`) | NOT NULL |
| `host` | `VARCHAR` | SMTP 主机 | NOT NULL |
| `port` | `INTEGER` | SMTP 端口 | NOT NULL |
| `secure` | `BOOLEAN` | 是否使用 SSL/TLS | DEFAULT TRUE |
| `auth_user` | `VARCHAR` | 认证用户名 | NOT NULL |
| `auth_pass_encrypted`| `VARCHAR` | 加密后的密码 | NOT NULL |
| `from_name` | `VARCHAR` | 发件人显示名称 | |
| `from_email` | `VARCHAR` | 发件人邮箱 | |
| `is_active` | `BOOLEAN` | 是否启用 | DEFAULT TRUE |
| `created_at` | `TIMESTAMP` | 创建时间 | DEFAULT NOW() |
| `updated_at` | `TIMESTAMP` | 更新时间 | DEFAULT NOW() |

### 3.7 邮件模板 (`email_templates`)
*存储动态邮件内容模板。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `SERIAL` | ID | PK |
| `type` | `VARCHAR` | 模板类型 (如 `verification_code`) | UNIQUE, NOT NULL |
| `subject` | `VARCHAR` | 邮件标题模板 | NOT NULL |
| `content` | `TEXT` | HTML 内容模板 | NOT NULL |
| `variables` | `JSONB` | 可用变量描述 | DEFAULT '[]'::jsonb |
| `updated_at` | `TIMESTAMP` | 最后更新时间 | DEFAULT NOW() |

### 3.8 邮件日志 (`email_logs`)
*记录邮件发送结果与错误信息。*

| 字段 | 类型 | 说明 | 约束 |
| :--- | :--- | :--- | :--- |
| `id` | `UUID` | 日志 ID | PK, DEFAULT gen_random_uuid() |
| `recipient` | `VARCHAR(255)` | 收件人 | NOT NULL |
| `template_type` | `VARCHAR(50)` | 模板类型 | NOT NULL |
| `subject` | `VARCHAR(255)` | 邮件标题 | |
| `status` | `VARCHAR(20)` | 发送状态 | NOT NULL |
| `error_message` | `TEXT` | 错误信息 | |
| `retry_count` | `INTEGER` | 重试次数 | DEFAULT 0 |
| `provider_id` | `INTEGER` | 服务商 ID | FK -> email_providers.id |
| `sent_at` | `TIMESTAMP` | 发送时间 | |
| `created_at` | `TIMESTAMP` | 创建时间 | DEFAULT NOW() |

### 3.9 验证码（已废弃）(`verification_codes`)
*说明：验证码已迁移到 Redis，此表仅作归档。*

## 5. 索引策略

*   `users(email)`: 唯一索引，加速登录查询。
*   `oauth_codes(code)`: 主键索引，加速授权码交换。
*   `oauth_codes(client_id, user_id)`: 组合索引，加速清理过期 Code。
*   `refresh_tokens(token_hash)`: 核心索引，用于 Token 验证与轮换。
*   `refresh_tokens(user_id)`: 加速用户所有 Token 的撤销。
*   `email_logs(status, created_at)`: 组合索引，便于统计与筛选。
