# 账户系统安全报告

> **版本**: V2.1
> **日期**: 2026-03-07
> **状态**: ✅ **现行基线 (Active Baseline)**

## 1. 当前安全基线

1. **认证协议升级**
   * 认证主链路采用 OAuth 2.0 Authorization Code + PKCE。
   * Token 签发算法为 RS256，并通过 `/.well-known/jwks.json` 对外发布公钥。
   * 发现文档由 `/.well-known/openid-configuration` 提供，供子系统自动发现端点。
2. **重定向与客户端校验**
   * `redirect_uri` 对 `allowed_origins` 执行严格匹配（Strict Match），禁止前缀匹配绕过。
   * `client_id` 必须已注册且处于启用状态，未授权客户端直接拒绝。
3. **令牌安全策略**
   * Access Token 短期有效，Refresh Token 使用轮换策略（Rotation）。
   * 旧 Refresh Token 在刷新后立即作废，阻断重放攻击。
   * 登出流程支持 Token 黑名单（Redis）失效控制。
4. **应用层防护**
   * 请求参数通过 Zod + `validate` 中间件统一校验。
   * 认证相关接口启用限流（`express-rate-limit`）。
   * 邮件模板渲染链路使用 DOMPurify 清洗，降低 XSS 风险。

## 2. 安全实现映射

| 模块 | 关键实现 | 说明 |
| :--- | :--- | :--- |
| 认证鉴权 | `middlewares/auth.js` | 统一解析 Bearer/Cookie 并注入 `req.user` |
| 角色授权 | `middlewares/roleCheck.js` | 管理员权限拦截 |
| 输入校验 | `middlewares/validate.js`, `validators/authSchema.js` | 统一参数 Schema 校验 |
| 频率限制 | `middlewares/rateLimit.js` | 全局与敏感接口限流 |
| OIDC 核心 | `services/oidcProvider.js`, `routes/ssoRoutes.js` | 处理 `/.well-known/*` 与 `/oauth/*` |
| 历史兼容 | `controllers/ssoController.js` | 仅保留废弃接口提示，不承担主认证链路 |
| Token 工具 | `utils/token.js` | RS256 签发与验签辅助 |
| 黑名单 | `utils/redis.js` | Logout 场景 Token 失效控制 |

## 3. 兼容项与收敛方向

1. **HS256 兼容回退**
   * 当前在 `verifyAccessToken` 中保留 RS256 失败后的 HS256 历史兼容回退，仅用于平滑过渡。
2. **收敛目标**
   * 当历史令牌全部自然过期后，移除 HS256 回退分支，统一收敛到 RS256 + JWKS。

## 4. 本轮审查结论

*   安全文档已与 V2.1 认证实现、路由行为和中间件职责重新对齐。
*   旧版 `getSsoToken` 描述已剔除，不再将其作为现行能力说明。
*   当前未发现需要新增归档处理的安全类活跃文档。
