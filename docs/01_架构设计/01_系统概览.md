# 账户系统架构概览

> **版本状态**: ✅ **Active (V2.0)**
> **最后更新**: 2026-02-26
> **基准环境**: 腾讯云（Ubuntu 22.04 LTS）
> **部署脚本**: `deploy_remote.ps1`（PowerShell 自动化）

## 1. 核心架构：四合院项目矩阵

本系统（Account System）作为整个产品矩阵的 **数字基座**，负责统一身份认证（SSO）、权限控制（RBAC）和数据中台服务。

### 1.1 拓扑结构

我们摒弃了过度设计的微服务架构，回归至最稳健的单体分层架构，但引入了 **Redis** 缓存与消息队列以支持高性能异步任务（如邮件服务）。

```mermaid
graph TD
    User[用户/客户端] -->|HTTPS/443| Nginx[前端网关 (account-frontend)]
    Nginx -->|/api/*| Node[后端 API (account-backend)]
    Nginx -->|/*| React[静态资源 (SPA)]
    Node -->|TCP/5432| DB[(核心数据库 iasbt-postgres)]
    Node -->|TCP/6379| Redis[(缓存/队列 Redis)]
    
    subgraph "Docker 内部网络（correction_default）"
        Node
        DB
        Redis
        PgAdmin[数据库管理 (pgadmin)]
    end
```

### 1.2 容器清单

| 容器名 | 镜像/构建源 | 端口映射 | 职责描述 |
| :--- | :--- | :--- | :--- |
| **iasbt-postgres** | `postgres:14-alpine` | 5432:5432 | **唯一真理数据库**。存储所有用户、日志及配置数据。 |
| **account-backend** | `node:20-alpine` | 3000:3000 | **业务核心**。Express.js 框架，提供 RESTful API 与 Email Worker。 |
| **account-frontend** | `nginx:alpine` | 80:80 | **流量入口**。托管 React/Vite 产物，反向代理 API 请求。 |
| **redis** | `redis:alpine` | 6379:6379 | **高速缓存**。用于 BullMQ 邮件队列与 Session 存储。 |
| **pgadmin** | `dpage/pgadmin4` | 8888:80 | **数据库管理**。可视化 SQL 操作面板。 |
| **portainer** | `portainer/portainer-ce` | 9000:9000 | **运维面板**。用于可视化容器管理。 |

> ⚠️ **严禁出现**：`nginx-gateway`, `postgres-business`, `postgrest` 等已废弃容器。

## 2. 核心功能模块

### 2.1 统一身份认证（SSO V2.0 - 纯 JWT 版）
- **多应用支持**: 通过 `public.applications` 表动态管理接入应用（如 Gallery, Toolbox）。
- **Token 策略**:
    - **Standard**: 标准 JWT (HS256)，用于所有接入应用。详见 [02_JWT认证指南](../02_开发指南/02_JWT认证指南.md)。
    - **Legacy (Supabase)**: 🚫 **已彻底废弃**。严禁使用。迁移请参考 [04_迁移指南](../02_开发指南/04_迁移指南.md)。
- **流程**:
    1. 应用跳转 `/api/sso/issue?target=...`
    2. 校验用户登录态
    3. 查询应用注册表与回调白名单
    4. 生成 Token 并重定向回 `target`
    > 完整对接流程请参阅 [03_单点登录指南](../02_开发指南/03_单点登录指南.md)。

### 2.2 邮件服务（V2.0）
- **架构**: 生产者-消费者模型（BullMQ + Redis）。
- **特性**:
    - **可视化模板**: 支持 Mustache 语法（`{{code}}`），提供 HTML 代码编辑与实时预览。
    - **多服务商**: 支持 SMTP（QQ、Outlook、Resend）动态切换与负载均衡。
    - **审计日志**: 记录每一封邮件的发送状态、耗时与错误信息。
    - **统计图表**: 24 小时发送趋势图（Recharts）。

### 2.3 管理员门户
- **RBAC**: 基于 `is_admin` 字段的中间件拦截 (`middlewares/roleCheck.js`)。
- **功能**:
    - **用户管理**: 查看、编辑、删除用户。
    - **应用管理**: 一键接入新应用，生成 `app_id` 与 `secret`。
    - **系统监控**: 查看服务健康状态与版本信息。

### 2.4 安全加固（V1.8.6）
- **输入验证**: 全面引入 **Zod** Schema，对所有 API 输入参数进行严格校验，拒绝非法请求。
- **频率限制**: 集成 `express-rate-limit`，针对不同接口实施差异化限流策略：
    - **全局**: 50次/15分钟
    - **验证码**: 3次/1分钟（防止短信轰炸）
- **XSS 防护**: 管理后台引入 **DOMPurify**，对所有富文本内容进行清洗，彻底阻断 XSS 攻击。

### 2.5 认证模块职责拆分

| 控制器入口 | 服务层方法 | 说明 |
| :--- | :--- | :--- |
| `sendVerificationCode` | `authService.sendVerificationCode(email, type)` | 发送验证码邮件 |
| `register` | `authService.verifyCode(email, code)` → `authService.register({ name, email, password })` | 注册前先校验验证码 |
| `login` | `authService.login({ account, password })` | 登录与 Token 生成 |
| `adminLogin` | `authService.adminLogin({ account, password })` | 管理员登录与权限校验 |
| `getMe` | 无 | 直接返回 `req.user` |
| `resetPassword` | `authService.resetPassword({ email, password, code })` | 验证码校验后重置密码 |
| `changePassword` | `authService.changePassword(userId, { oldPassword, newPassword })` | 修改密码 |

### 2.7 数据层与注册表一致性

1.  **SSO 应用注册表** 与 **微应用运行状态表** 当前共用 `public.applications` 名称，但字段定义不同（`app_id/allowed_origins` vs `name/version/status`），部署时必须统一方案。
2.  **验证码存储** 已迁移到 Redis（`utils/verificationStore.js`），数据库 `verification_codes` 仅作历史兼容。

## 3. 部署与运维

### 3.1 自动化部署流水线
所有部署操作 **必须** 通过项目根目录下的 `deploy_remote.ps1` 脚本执行。

**执行命令**:
```powershell
.\deploy_remote.ps1 "v1.8.6: Security Hardening & Refactor"
```

**流水线步骤**:
1.  **本地提交**: 自动提交代码 (Git Commit & Push)。
2.  **远程拉取**: SSH 连接腾讯云，拉取最新代码 (Git Pull)。
3.  **构建镜像**: 在服务器端构建 Docker 镜像 (`docker compose build`)。
4.  **发布部署**: 重启服务 (`down` -> `up -d`)。
5.  **版本校验**: 自动执行 `/api/health` 检查，验证 Version 字段。
6.  **环境同步**: 自动将根目录 `.env` 复制到 `/home/ubuntu/account/deploy/correction/.env`。

### 3.2 目录规范
服务器端文件结构严格遵循：

```text
/home/ubuntu/account/
├── deploy/
│   └── correction/          <-- 部署核心目录 (Source of Truth)
│       ├── docker-compose.yml
│       ├── Dockerfile.api
│       ├── Dockerfile.web
│       ├── nginx.conf
│       └── servers.json     (PgAdmin Config)
├── server.js                <-- 后端入口
└── ...
```
