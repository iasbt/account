### 3.2 测试用例清单 (Test Cases)

#### 后端 (Backend API)
*   **OAuth 2.0 / OIDC**:
    *   `GET /.well-known/openid-configuration`: 发现文档格式正确。
    *   `GET /.well-known/jwks.json`: 公钥格式正确 (RS256)。
    *   `POST /api/oauth/authorize`:
        *   Valid: 返回 302 跳转 + Auth Code。
        *   Invalid: `redirect_uri` 不匹配 -> 400。
        *   PKCE: 缺少 `code_challenge` -> 400。
    *   `POST /api/oauth/token`:
        *   Exchange: Code + Verifier -> Access + Refresh Token。
        *   Refresh: Refresh Token -> New Access + New Refresh Token。
        *   Rotation: 旧 Refresh Token 再次使用 -> 拒绝并报警。
*   **Auth**:
    *   `POST /api/auth/login`: 账号密码登录。
    *   `GET /api/auth/me`: 验证 Token 解析。

#### 前端 (Frontend Components)
*   **Login Page**:
    *   输入框正常渲染。
    *   点击登录按钮触发 API 调用。
*   **SSO Redirect**:
    *   `LogoutPage` 清除本地存储并跳转。

## 4. 脚本模板 (Script Templates)

### 4.1 PKCE 流程验证脚本 (Node.js)

参考 `scripts/test_pkce.js`:

```javascript
// 模拟 PKCE 流程
const verifier = generateCodeVerifier();
const challenge = await generateCodeChallenge(verifier);

// 1. 获取授权码
const authRes = await axios.post('/api/oauth/authorize', {
  response_type: 'code',
  code_challenge: challenge,
  code_challenge_method: 'S256',
  ...
});

// 2. 换取 Token
const tokenRes = await axios.post('/api/oauth/token', {
  grant_type: 'authorization_code',
  code: authCode,
  code_verifier: verifier, // 必须携带 Verifier
  ...
});
```

### 4.2 Refresh Token 轮换验证

参考 `scripts/test_refresh_flow.js`:

```javascript
// 1. 使用 Refresh Token 刷新
const refreshRes = await axios.post('/api/oauth/token', {
  grant_type: 'refresh_token',
  refresh_token: oldRefreshToken
});
const newRefreshToken = refreshRes.data.refresh_token;

// 2. 尝试再次使用旧 Token (应失败)
try {
  await axios.post('/api/oauth/token', {
    grant_type: 'refresh_token',
    refresh_token: oldRefreshToken
  });
} catch (e) {
  assert(e.response.status === 401); // 预期结果
}
```
