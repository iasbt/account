# 开发与部署操作指南 (Development & Deployment Operation Guide)

> **文档版本**: v2.1
> **更新时间**: 2026-02-27
> **适用对象**: 开发人员 (Developers)

本手册用于指导 **本地开发环境搭建** 及 **自动化部署操作**。

---

## 1. 本地开发环境 (Local Development Environment)

### 1.1 基础环境 (Prerequisites)
*   **OS**: Windows 10/11 (推荐) or macOS/Linux
*   **Runtime**: Node.js 20+ (LTS)
*   **Package Manager**: npm 10+
*   **Shell**: PowerShell 5.x+ (Windows)

### 1.2 启动项目 (Start Project)

```powershell
# 1. 安装依赖 (Install Dependencies)
npm install

# 2. 启动数据库 (Start Database - Optional if using remote DB)
# 如果连接远程测试库，需配置 .env

# 3. 启动开发服务器 (Start Dev Server)
npm run dev
```

### 1.3 关于 RSA 密钥 (About RSA Keys)
*   **本地开发 (Local)**: 未提供 `certs/private.pem` / `certs/public.pem` 时，系统会临时生成内存 RSA 密钥并继续使用 **RS256**。
*   **生产环境 (Production)**: 必须使用持久化 RSA 密钥对（详见 `01_部署手册.md`），避免重启后密钥变化导致历史 Token 失效。

## 2. 自动化部署 (Automated Deployment)

**严禁 (Forbidden)** 手动上传文件或直接在服务器修改代码。必须使用自动化脚本。

### 2.1 部署命令 (Deployment Command)

在 VSCode 终端 (PowerShell) 执行：

```powershell
# 格式: .\deploy_remote.ps1 "Commit Message"
.\deploy_remote.ps1 "v2.1: Update feature X"
```

### 2.2 脚本执行流 (Execution Flow)
1.  **Local**: Git Commit & Push（脚本自动执行 `git add/commit/push`）。
2.  **Remote**: Git Fetch + Hard Reset（同步 `origin/main`）。
3.  **Remote**: 同步 `.env` 到 `deploy/correction`，并检查/生成持久化 RSA 密钥。
4.  **Remote**: `docker compose up -d --build --remove-orphans`（构建并更新容器）。
5.  **Remote**: 版本校验（优先 `/health`，失败回退 `/api/health`，并比对 `package.json` 版本）。

---

## 3. 域名解析配置 (DNS & EdgeOne)

如果涉及新域名接入，请登录腾讯云 EdgeOne 控制台配置：

1.  **DNS 记录**: 添加 CNAME 指向 EdgeOne 加速域名。
2.  **回源协议**: 必须设置为 **HTTP** (因为服务器 Nginx 监听 80 端口)。
3.  **SSL 证书**: 由 EdgeOne 自动托管，无需在服务器配置证书。

## 4. 常用维护命令 (Maintenance)

```powershell
# 查看本地运行状态
npm run dev
```

### 4.1 验证 OIDC 发现文档 (Verify OIDC Discovery)
访问 `http://localhost:3000/.well-known/openid-configuration`，应返回 JSON 格式的 OIDC 配置信息。

### 4.2 验证公钥 (Verify JWKS)
访问 `http://localhost:3000/.well-known/jwks.json`，应返回 RS256 公钥信息。

### 4.3 验证 PKCE 强制要求 (Verify PKCE Requirement)
从 V1.9.18 开始，对于公共客户端（无 client_secret），强制要求使用 PKCE (S256)。
尝试不带 `code_challenge` 的授权请求应返回错误。

### 4.4 结构化日志 (Structured Logging)
所有关键操作（认证、SSO、错误）现在通过 `utils/logger.js` 输出结构化 JSON 日志。
生产环境建议配合日志聚合工具（如 ELK 或 CloudWatch）使用。

### 4.5 集成测试 (Integration Tests)
集成测试 (E2E Tests) 依赖于专用的 Docker 测试环境（PostgreSQL + Redis）。

1.  **准备环境**: 启动 `docker-compose.test.yml` (如果存在) 或手动启动对应服务。
    -   Postgres Test DB: Port 5433, User `postgres`, Password `password`, DB `account_test`.
    -   Redis Test: Port 6380.
2.  **执行测试**:
    ```powershell
    # Windows PowerShell
    $env:RUN_INTEGRATION_TESTS="true"; npm run test:integration
    ```
    如果不设置环境变量，测试将被跳过。
