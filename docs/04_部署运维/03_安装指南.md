# 本地开发环境搭建指南

本手册用于指导 **通过自动化脚本完成部署**，禁止手动在服务器上改文件。

## 1. 准备工作

在本地开发机确认以下条件：
- 已安装 Git、Node.js 20+、PowerShell 5.x
- 项目根目录存在 `deploy/correction/`（包含 `docker-compose.yml`、`Dockerfile.api`、`Dockerfile.web`、`nginx.conf`）
- **机密信息不入库**：仅允许修改 `.env.example`，真实 `.env` 通过服务器端或 Docker Env 注入

## 2. 部署方式（唯一允许）

**禁止** 手动上传与手动登录服务器修改文件。  
**必须** 使用自动化脚本：`deploy_remote.ps1`。

### 执行命令 (PowerShell)

```powershell
.\deploy_remote.ps1 "v1.8.6: Deploy"
```
脚本会自动执行：
1. 本地 Git 提交与推送
2. 远程拉取最新代码
3. 进入 `/home/ubuntu/account/deploy/correction/`
4. Docker 构建与启动
5. `/api/health` 版本校验

## 3. 域名解析 (EdgeOne 配置)

登录腾讯云 EdgeOne 控制台：
1.  找到 `www.iasbt.cloud` 域名。
2.  添加 DNS 记录：
    *   `www.iasbt.cloud` -> CNAME 指向 EdgeOne 提供的加速域名。
    *   或者分别添加 `account`, `www` 指向您的服务器 IP。
    *   **重要**：在 EdgeOne 的“回源配置”中，设置 **回源协议** 为 **HTTP**（SSL 由 EdgeOne 负责）。

## 4. 验证访问

打开浏览器访问：
*   `https://account.iasbt.com` -> 前端页面可访问
*   `https://account.iasbt.com/api/health` -> `status=ok` 且版本号与 `package.json` 一致

## 5. 后续维护

*   **查看日志**：`docker logs -f account-backend`
*   **重启服务**：`docker restart account-backend`
*   **备份数据**：通过数据库侧 `pg_dump` 或运维脚本完成
