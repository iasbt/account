## 3. 部署前置准备 (Pre-deployment Preparation)

### 3.1 生成 RSA 密钥对 (Generate RSA Keys)
**重要**：自 V1.9.0 起，系统启用 RS256 非对称加密。必须在服务器上手动生成密钥对并挂载到容器中。

在服务器执行以下命令：
```bash
# 进入部署目录
cd /home/ubuntu/account/deploy/correction/

# 创建证书目录
mkdir -p certs

# 生成私钥 (Private Key)
openssl genrsa -out certs/private.pem 2048

# 生成公钥 (Public Key)
openssl rsa -in certs/private.pem -pubout -out certs/public.pem

# 设置权限 (Permissions)
chmod 600 certs/private.pem
chmod 644 certs/public.pem
```

## 4. 自动化部署脚本 (`deploy_remote.ps1`)

脚本位置: `c:\My_Project\account\deploy_remote.ps1`

### 4.1 功能详解 (Function Details)
此脚本是部署的核心，执行以下原子操作 (Atomic Operations)：
1.  **SSH 连接**: 使用密钥连接服务器。
2.  **代码同步**: 远程执行 `git fetch origin main` + `git reset --hard origin/main`。
3.  **编排目录**: 进入 `/home/ubuntu/account/deploy/correction/`。
4.  **服务编排**: `docker compose build` 与 `docker compose up -d`。
5.  **版本校验**: 优先调用 `/health`，失败时回退 `/api/health` 并校验版本号。

### 4.2 常用运维命令 (Common Operations)
在 VSCode 终端中运行：

```powershell
# 1. 一键部署（会重启服务） - Deploy
.\deploy_remote.ps1 "v2.1: Deploy with RS256 & PKCE"
```

## 6. 故障排查与修复记录 (Troubleshooting & Fix Log)

### [2026-02-27] V2.1 升级记录
*   **现象**: `deploy_remote.ps1` 部署后 Token 验证失败。
*   **原因**: 缺少 RSA 密钥对 (`certs/private.pem`)。
*   **修复**: 在服务器手动生成密钥对，并修改 `docker-compose.yml` 挂载 `./certs:/app/certs`。

### [2026-02-25] V1.8.6 修复记录
*   **现象**: `deploy_remote.ps1` 校验 `/api/health` 失败。
*   **原因**: 控制器版本号未递增或健康检查未同步。
*   **修复**: 统一更新 `package.json` 与 `healthController.js` 的版本号。

## 7. 后续配置清单 (Configuration Checklist)

### 7.1 腾讯云 EdgeOne 配置
由于服务器 Nginx 监听 80 端口，必须调整 EdgeOne 回源策略：
- [ ] 进入 EdgeOne 控制台 -> `iasbt.cloud` / `account.iasbt.cloud` / `logto.iasbt.cloud` / `logto-console.iasbt.cloud`。
- [ ] **DNS 解析**：添加 `*.iasbt.cloud` 与根域 `iasbt.cloud` 记录并指向 EdgeOne。
- [ ] **回源配置**：将 **回源协议 (Origin Protocol)** 设置为 **HTTP** (重要！否则 502)。
