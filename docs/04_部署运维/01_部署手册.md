| 服务组件 | 容器名称 | 端口映射 | 版本 | 说明 |

- **公网入口**: 腾讯云 EdgeOne (CDN/WAF) -> Nginx (80 端口)
- **内部转发**: Nginx -> `account-backend` -> `iasbt-postgres`
- **SSL/HTTPS**: 由 EdgeOne 托管证书，回源协议为 **HTTP**。
 - **网络**: `correction_default`

## 3. 自动化部署脚本 (`deploy_remote.ps1`)

脚本位置: `c:\My_Project\account\deploy_remote.ps1`

### 3.1 功能详解
此脚本是部署的核心，执行以下原子操作：
1.  **SSH 连接**: 使用密钥连接服务器。
2.  **代码同步**: 远程仓库 `git pull` 拉取最新代码。
3.  **编排目录**: 进入 `/home/ubuntu/account/deploy/correction/`。
4.  **服务编排**: `docker compose build` 与 `docker compose up -d`。
5.  **版本校验**: 调用 `/api/health` 校验版本号。

### 3.2 常用运维命令
在 VSCode 终端中运行：

```powershell
# 1. 一键部署（会重启服务）
.\deploy_remote.ps1 "v1.8.6: Deploy"
```

## 4. 故障排查与修复记录

### [2026-02-25] 关键修复记录

#### 1. 版本不一致导致部署失败
*   **现象**: `deploy_remote.ps1` 校验 `/api/health` 失败。
*   **原因**: 控制器版本号未递增或健康检查未同步。
*   **修复**: 统一更新 `package.json` 与 `healthController.js` 的版本号。

## 5. 后续配置清单

### 5.1 腾讯云 EdgeOne 配置
由于服务器 Nginx 监听 80 端口，必须调整 EdgeOne 回源策略：
- [ ] 进入 EdgeOne 控制台 -> `img.iasbt.com`。
- [ ] **DNS 解析**：添加 `*.iasbt.com` CNAME 指向 EdgeOne。
- [ ] **回源配置**：将 **回源协议 (Origin Protocol)** 设置为 **HTTP** (重要！否则 502)。

## 6. 备份与迁移
*   **备份**: 通过 `pg_dump` 导出数据库，再同步到安全存储。
*   **恢复**: 在新服务器导入数据库后执行 `docker compose up -d`。
