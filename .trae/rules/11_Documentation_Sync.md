# Documentation Sync Standard（文档同步规范）

> Status: Active  
> Effective Date: 2026-02-24  
> Enforcement: ✅ Strict（强制执行）

## 1. 适用范围
- 所有影响测试环境或生产环境的代码提交（功能、性能、配置、接口、依赖等）。

## 2. 提交前必做
- 在每次提交 / 推送代码前，必须先将本次版本迭代中涉及的所有功能更新、性能优化、缺陷修复、配置变更、接口调整、依赖升级等变更内容，按照项目文档管理规范，完整、准确、及时地同步更新到 readme 文件夹对应的文档中。

## 3. 一致性要求
- 所有相关文档必须对齐：版本号、变更描述、责任人、时间戳等关键信息。
- 严禁出现“代码已变更但文档未更新”或“不同文档之间版本与变更记录不一致”的情况。

## 4. 审核与阻断
- 代码评审与合并前必须检查文档是否已按本规则更新。
- 如未满足本规则，应视为评审未完成，不得合并、不得部署。
