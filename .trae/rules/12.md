# Account System V1.6 (Frozen) - Collaboration Red Lines

> **Status**: 🔴 **RED LINES (Strictly Enforced)**
> **Context**: Complex Environment Alignment
> **Violation**: STRICTLY FORBIDDEN

## 1. Path Absolutization (路径绝对化)
*   **Rule**: 禁止使用相对路径。
*   **Requirement**: 必须明确指出绝对路径。
*   **Example**: `/home/ubuntu/account/deploy/correction/`

## 2. Deployment Atomicity (部署原子化)
*   **Git Flow**: `Local Edit` -> `git commit` -> `git push` -> `Remote Pull`.
*   **Prohibition**: 🚫 **禁止** 跳过 GitHub 直接修改服务器文件。
*   **Atomicity**: 部署必须通过版本控制系统同步，保持本地与远程一致。

## 3. Environment Differentiation (环境差异化)
*   **Local**: 仅用于代码逻辑编写和简单的 `npm test`。
*   **Prod**: 必须且仅能运行在 **Docker 容器** 中。
*   **Consistency**: 生产环境严禁运行裸 Node.js 进程。

## 4. Config Isolation (配置隔离)
*   **Secret**: 🚫 **禁止** 将服务器的真实 `.env` 推送到 Git。
*   **Action**: Trae 只能修改 `.env.example`。
*   **Security**: 敏感配置必须在服务器端手动管理或通过安全通道注入。

## 5. Version Strong Verification (版本强校验)
*   **Trigger**: 任何针对后端的修改，**必须** 同时修改 `version` 变量。
*   **Success Metric**: 部署后访问 `/api/health`。
*   **Failure Condition**: 若未返回新版本号，即视为 **部署失败** (Rollback required).
