一、Logto 文件清单（与账户项目隔离）

所有 Logto 资源固定放在 `deploy/correction/logto/`，不改动账户项目原有后端与前端目录。

| 文件 | 作用 | 隔离策略 |
|---|---|---|
| `docker-compose.yml` | Logto 与 PostgreSQL 编排 | 容器名独立、端口仅绑定 `127.0.0.1`、独立 Docker 网络 |
| `.env` | 运行参数与密钥 | Logto 专属参数，不复用账户项目 `.env` |
| `deploy-logto.sh` | 本地/远程部署脚本 | 阻止默认共享网络，避免误接入账户网络 |
| `check-logto.sh` | 健康检查脚本 | 仅检查 Logto 自身容器与接口 |

二、当前已落地的隔离规则

1. 网络隔离
- 默认网络：`DOCKER_NETWORK=logto-isolated`
- 若设置为 `correction_default`，脚本会阻止部署
- 仅当明确设置 `LOGTO_ALLOW_SHARED_NETWORK=1` 才允许共享网络

2. 端口隔离
- `3001` 与 `3002` 仅绑定本机回环地址
- 映射形式：`127.0.0.1:${LOGTO_PORT}:3001`、`127.0.0.1:${LOGTO_ADMIN_PORT}:3002`
- 不直接暴露公网端口，统一通过 Nginx 反向代理

3. 数据隔离
- 默认数据目录：`./logto-data`
- 生产建议目录：`/data/logto`
- `.gitignore` 已忽略 `deploy/correction/logto/.env` 与 `deploy/correction/logto/logto-data/`

4. 配置隔离
- 容器名独立：`logto-core`、`logto-postgres`
- 组合名独立：`COMPOSE_PROJECT_NAME=logto`
- 密钥与数据库密码仅存在 Logto 子目录 `.env`

三、一键执行命令（TRAE 终端）

```bash
cd deploy/correction/logto
bash deploy-logto.sh local
```

```bash
cd deploy/correction/logto
bash deploy-logto.sh remote
```

```bash
cd deploy/correction/logto
bash check-logto.sh
```

四、生产环境建议

1. 将 `LOGTO_DATA_DIR` 改为 `/data/logto`
2. 将 Logto 域名交给 Nginx 反向代理到 `127.0.0.1:3001`
3. 替换生产密钥后再上线（`LOGTO_ENCRYPTION_KEY`、`POSTGRES_PASSWORD`）

五、与账户项目并行运行原则

1. 不修改账户项目 `deploy/correction/docker-compose.yml`
2. 不复用账户项目 `.env` 与数据库容器
3. 不占用账户项目核心端口（3000、5432）
4. 新增能力只在 `deploy/correction/logto/` 内维护
